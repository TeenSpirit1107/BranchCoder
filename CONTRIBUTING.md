# Contributing Guide

- [English](#english-version)/[中文](#中文版本)

## English Version

Thank you for contributing to this project.  
To keep the commit history clean and consistent, please follow the convention below.

---

### Commit Message Convention

All commit messages **must start with one of the allowed prefixes** and follow this format:

```
[prefix] short description
```

### Allowed Prefixes

| Prefix | Description |
|------|-------------|
| `[feature]` | New features or functional changes |
| `[bugfix]`  | Bug fixes |
| `[doc]`     | Documentation changes |
| `[chore]`   | Maintenance tasks (configs, dependencies, tooling, etc.) |

---

Thank you for your contribution!

## 中文版本

感谢你对本项目的贡献！为了保持提交历史清晰、统一，请在提交代码时遵循以下规范。

---

### Commit Message 规范

本项目要求所有 commit message **必须以指定前缀开头**，格式如下：

```
[前缀] 提交说明
```

### 允许的前缀

| 前缀 | 含义 |
|----|----|
| `[feature]` | 新功能或功能性改动 |
| `[bugfix]`  | 修复 bug |
| `[doc]`     | 文档相关改动（README、注释等） |
| `[chore]`   | 杂项改动（构建脚本、配置、依赖、工具等） |

---

感谢你的配合与贡献！
